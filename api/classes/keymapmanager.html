add

  • add(bindings: IKeyBinding[]): IDisposable
    • Defined in manager.ts:120

    Add key bindings to the key map manager.

    Parameters

    • bindings: IKeyBinding[]

      The key bindings to add to the manager.

    Returns IDisposable

    A disposable which removes the added key bindings.

    Notes

    If a key binding is invalid, a warning will be logged to the console and the offending key binding will be ignored.

    If multiple key bindings are registered for the same sequence, the binding with the highest CSS specificity is executed first. Ties in specificity are broken based on the order in which the key bindings are added to the manager, with newer bindinding taking precedence.

    Ambiguous key bindings are resolved with a timeout.

processKeydownEvent

  • processKeydownEvent(event: KeyboardEvent): void
    • Defined in manager.ts:141

    Process a 'keydown' event and invoke a matching key binding.

    Parameters

    • event: KeyboardEvent

      The event object for a 'keydown' event.

      Notes

      This should be called in response to a 'keydown' event in order to invoke the handler function of the best matching key binding.

      The manager does not install its own key event listeners. This allows user code full control over the nodes for which the manager processes 'keydown' events.

    Returns void
